# Деплой в облако (dockhost.ru)

Цель: оба аккаунта работают 24/7 в облаке, компьютер можно выключать. Доставку делает
**один общий бот**: перехваты с каждого аккаунта приходят его владельцу (`OWNER_ID`) в личку,
поэтому у тебя и у друга в одном боте — только свои уведомления. Сайта и веб-адреса нет.

Ключевая идея: **код** едет в облако из Git-репозитория (без секретов), а **секреты и сессии**
(`.env`, `.env.friend`, `.env.bot`, папки сессий) лежат на **примонтированном диске** `/data` и
задаются уже в облаке. Так ключи не попадают в репозиторий/образ.

---

## Шаг 1. Залить код в Git-репозиторий
`.gitignore` уже исключает `.env*` и `data*/`, поэтому секреты не утекут.

```powershell
cd "c:\Users\timur\OneDrive\Документы\Diaslog Spy bot"
git add .
git commit -m "..."
git push
```
Проверь на сайте репозитория, что `.env`, `.env.friend`, `data/`, `data-friend/` **НЕ** загрузились.
Репозиторий лучше держать **приватным** (см. раздел «Безопасность»).

## Шаг 2. Создать контейнер на dockhost.ru
1. Зарегистрируйся на https://dockhost.ru, создай **проект**.
2. Создай **контейнер**, источник — **сборка из Git-репозитория** (укажи свой репо,
   для приватного добавь токен доступа). Dockerfile в корне уже есть — соберётся сам.
3. Публиковать порт **не нужно** — веб-сервера нет, контейнер просто держит ботов онлайн.

## Шаг 3. Добавить диск (постоянное хранилище)
В настройках контейнера добавь **том/диск** и примонтируй его в путь **`/data`**
(размер 1–2 ГБ хватит; медиа чистятся по `RETENTION_DAYS`).

## Шаг 4. Переменные окружения
Задай в панели контейнера одну переменную:
```
DIASLOG_DATA_DIR=/data
```
Она говорит, что и конфиги (`.env`), и сессии берутся с примонтированного диска `/data`.

## Шаг 5. Положить конфиги и войти в аккаунты (одноразово)
Контейнер запустится, но боты будут «без сессии», пока не сделаешь вход.

1. Открой у контейнера **консоль/терминал** (в панели dockhost) — это shell внутри контейнера.
2. Создай конфиги на диске `/data`. Бот **один** (`.env.bot`), у каждого аккаунта только свой
   `OWNER_ID`. **Значения возьми из своих локальных файлов** (НЕ из репозитория — там их нет).
   Подставь вместо `...`:
   ```sh
   cd /data
   cat > .env.bot <<'EOF'
   BOT_TOKEN=...
   API_ID=...
   API_HASH=...
   ADMIN_ID=...
   EOF
   cat > .env <<'EOF'
   API_ID=...
   API_HASH=...
   OWNER_ID=...
   CACHE_MEDIA=true
   RETENTION_DAYS=7
   EOF
   cat > .env.friend <<'EOF'
   API_ID=...
   API_HASH=...
   OWNER_ID=...
   CACHE_MEDIA=true
   RETENTION_DAYS=7
   EOF
   ```
   > `OWNER_ID` в `.env` — это **ты**, в `.env.friend` — **друг**. Каждому придут перехваты
   > только с его аккаунта. Узнать id: написать общему боту `/start` — он пришлёт число.
   > `ADMIN_ID` в `.env.bot` — твой id: тебе откроется «🛠 Админ-панель» в боте, где можно
   > добавлять/удалять отслеживаемые аккаунты (вход по телефону прямо в чате).
3. Войди в каждый аккаунт (введёшь номер и код из Telegram):
   ```sh
   cd /app
   python main.py            # твой аккаунт
   python main.py friend     # аккаунт друга (код придёт другу — он продиктует)
   ```
   Сессии сохранятся в `/data/default` и `/data/friend` на постоянном диске.
   > Вход с нового IP (дата-центр) — Telegram может разок запросить код. Это нормально.

## Шаг 6. Перезапустить
Перезапусти контейнер — `run.py` сам поднимет все аккаунты с сессией. В личке от каждого
бота начнут приходить перехваты (удалённые / изменённые / одноразовые). Компьютер можно
выключать. ✅

---

## Обновление кода потом
Запушил изменения в репозиторий → в панели dockhost нажми **Пересобрать/Redeploy**.
Диск `/data` (сессии, кэш) сохраняется между пересборками.

## Безопасность
- Сессии в `/data` = доступ к аккаунтам. Держи проект/панель под надёжным паролем и 2FA.
- Репозиторий держи **приватным**: код раскрывает, как устроен инструмент перехвата.
- Наружу ничего не публикуется — перехваты идут только в личку владельцу (`OWNER_ID`).
