# Деплой в облако (dockhost.ru)

Цель: сайт + оба бота работают 24/7 в облаке, компьютер можно выключать.

Ключевая идея: **код** едет в облако из Git-репозитория (без секретов), а **секреты и сессии**
(`.env`, `.env.friend`, папки сессий) лежат на **примонтированном диске** `/data` и задаются уже
в облаке. Так ключи не попадают в репозиторий/образ.

---

## Шаг 1. Залить код в приватный Git-репозиторий
`.gitignore` уже исключает `.env*` и `data*/`, поэтому секреты не утекут.

```powershell
cd "c:\Users\timur\OneDrive\Документы\Diaslog Spy bot"
git init
git add .
git commit -m "Diaslog Spy"
```
Создай **приватный** репозиторий на GitHub/GitLab и запушь:
```powershell
git remote add origin <URL-приватного-репо>
git push -u origin main
```
Проверь на сайте репозитория, что `.env`, `.env.friend`, `data/`, `data-friend/` **НЕ** загрузились.

## Шаг 2. Создать контейнер на dockhost.ru
1. Зарегистрируйся на https://dockhost.ru, создай **проект**.
2. Создай **контейнер**, источник — **сборка из Git-репозитория** (укажи свой приватный репо,
   при необходимости добавь токен доступа). Dockerfile в корне уже есть — он соберётся сам.
3. Порт приложения — **8000**. Включи публикацию HTTP — dockhost выдаст адрес вида
   `https://<имя>.dockhost.ru` с HTTPS.

## Шаг 3. Добавить диск (постоянное хранилище)
В настройках контейнера добавь **том/диск** и примонтируй его в путь **`/data`**
(размер 1–2 ГБ хватит; медиа чистятся по `RETENTION_DAYS`).

## Шаг 4. Переменные окружения
Задай в панели контейнера:
```
DIASLOG_HOST=0.0.0.0
DIASLOG_DATA_DIR=/data
DIASLOG_PASSWORD=придумай_надёжный_пароль
DIASLOG_PORT=8000
```

## Шаг 5. Положить конфиги и войти в аккаунты (одноразово)
Контейнер запустится, но боты будут «без сессии», пока не сделаешь вход.

1. Открой у контейнера **консоль/терминал** (в панели dockhost) — это shell внутри контейнера.
2. Создай конфиги на диске `/data`. **Значения возьми из своих локальных файлов**
   `.env` и `.env.friend` (НЕ из репозитория — там их нет). Подставь вместо `...`:
   ```sh
   cd /data
   cat > .env <<'EOF'
   API_ID=...
   API_HASH=...
   BOT_TOKEN=...
   OWNER_ID=...
   CACHE_MEDIA=true
   RETENTION_DAYS=7
   EOF
   cat > .env.friend <<'EOF'
   API_ID=...
   API_HASH=...
   BOT_TOKEN=...
   OWNER_ID=...
   CACHE_MEDIA=true
   RETENTION_DAYS=7
   EOF
   ```
3. Войди в каждый аккаунт (введёшь номер и код из Telegram):
   ```sh
   cd /app
   python main.py            # твой аккаунт
   python main.py friend     # аккаунт друга (код придёт другу — он продиктует)
   ```
   Сессии сохранятся в `/data/default` и `/data/friend` на постоянном диске.
   > Вход с нового IP (дата-центр) — Telegram может разок запросить код. Это нормально.

## Шаг 6. Перезапустить и открыть
Перезапусти контейнер (или он сам подхватит сессии). Открой `https://<имя>.dockhost.ru`,
браузер спросит логин/пароль — введи любой логин и `DIASLOG_PASSWORD`.

В ленте появятся перехваты, аккаунты будут «Активны». Компьютер можно выключать. ✅

---

## Обновление кода потом
Запушил изменения в репозиторий → в панели dockhost нажми **Пересобрать/Redeploy**.
Диск `/data` (сессии, ленты) сохраняется между пересборками.

## Безопасность
- Дашборд **всегда** под паролем (`DIASLOG_PASSWORD`) — это твои личные переписки.
- Сессии в `/data` = доступ к аккаунтам. Держи проект/панель под надёжным паролем и 2FA.
- Репозиторий — **приватный**.
