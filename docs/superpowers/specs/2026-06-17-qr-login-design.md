# Вход в аккаунт по ссылке/QR вместо кода

Дата: 2026-06-17

## Цель

Заменить вход добавляемого аккаунта (сейчас «телефон → код») на **вход по
login-токену Telegram**: бот присылает тапабельную ссылку `tg://login?token=…`
(основное) и тот же токен QR-картинкой (запасное, для второго устройства).
Владелец подтверждает вход со своего телефона — **без SMS/кодов**. Это обходит
проблему «Telegram не доставляет код стороннему api_id» и делает добавление
«как у Dialog Spy»: нажал ссылку → подтвердил.

## Почему так

- Прочитать личку/удалённые можно только имея доступ к аккаунту → нужен вход
  (юзербот-сессия). Но вход можно сделать **по login-токену**, а не по коду.
- `tg://login?token=…`, тапнутый на телефоне залогиненного аккаунта, открывает
  «Подтвердить вход?» — подтверждение и есть «кнопка». Кода нет.
- QR того же токена — для случая «добавляю с другого устройства» (показал QR на
  одном экране, отсканировал телефоном со второго).

## Поток (UX)

1. 🛠 Админ-панель → ➕ Добавить аккаунт.
2. Бот шлёт **одно сообщение**: фото QR + подпись (HTML):
   - «На **этом** телефоне — нажми **✅ Подтвердить вход**» (ссылка `tg://login?token=…`);
   - «С **другого** устройства — отсканируй QR: Настройки → Устройства →
     Подключить устройство»;
   - «Ссылка обновляется автоматически (~30 сек)».
   Кнопка **Отмена** (`admin:cancel`).
3. Владелец тапает ссылку (или сканирует QR) → подтверждает вход в Telegram.
4. Если 2FA — бот просит пароль облака (один раз, текстом).
5. Финал: «✅ Аккаунт N добавлен», слежка поднимается.

## Технически (Telethon)

- `qr = await client.qr_login()` → `qr.url` = `tg://login?token=…`.
- Рендер QR: `qrcode.make(qr.url)` → PIL-картинка → PNG в `io.BytesIO` (с `.name`).
- Отправка: `bot.send_file(admin_id, png, caption=..., parse_mode="html",
  buttons=cancel)`; сохраняем объект сообщения для обновления.
- **Обновление токена:** фоновая asyncio-задача в цикле:
  - `await qr.wait(timeout=30)`:
    - успех (вернулся user) → финал;
    - `SessionPasswordNeededError` → шаг "password", просим пароль, выходим из цикла;
    - `asyncio.TimeoutError` → `await qr.recreate()` → перерисовать QR и
      **отредактировать то же сообщение** (новый QR + новая ссылка);
  - общий лимит ~180 сек; вышел — «QR истёк, нажми ➕ заново», отключаем клиент.
- Пароль 2FA: на шаге "password" `await client.sign_in(password=pw)` → финал.
- **Финал (как сейчас, переиспользуем `_finalize`):** `get_me()` → `name=id<id>`,
  `owner_id=id`, `write_profile_env`, `Profile`, перенос сессии (+journal/wal),
  `start_profile`.

## Архитектура

### `admin.py` (переписать визард)
- `Wizard`: поля `client`, `qr` (QRLogin), `qr_msg` (сообщение с картинкой),
  `qr_task` (фоновая задача), `step` ∈ {"qr", "password"}.
- `begin_add(admin_id)`: cancel прошлого → создать клиент, connect →
  `client.qr_login()` → отправить QR-сообщение → запустить фоновую `qr_task`
  (`_qr_loop`). Возвращает None (текст не нужен — всё в сообщении с QR).
- `_qr_loop(admin_id, wiz)`: цикл ожидания/обновления (выше). Шлёт финал/ошибки
  напрямую через `self.bot`.
- `_render_qr(url) -> BytesIO`: PNG из `qr.url`.
- `feed_message(admin_id, text)`: оставляем **только шаг "password"**.
- `cancel(admin_id)`: отменить `qr_task`, disconnect клиента, удалить временные
  файлы сессии (`.login_<id>.session` + journal/wal/shm).
- Удалить `extract_code` и `tests/test_admin.py` (кодов больше нет; QR/Telethon
  юнит-тестам не поддаётся — проверяется вручную).

### `run.py`
- Почти без изменений. `handle_admin` ветка `add` теперь вызывает `begin_add`
  (которая сама шлёт QR-сообщение) и не редактирует экран в текст-приглашение.
  Достаточно `await _safe_answer(event)` — само сообщение с QR придёт отдельно.
- `on_message` остаётся (шаг пароля).

### Зависимости
- `requirements.txt`: добавить `qrcode[pil]`. Dockerfile уже ставит requirements.

### `login_local.py`
- Перевести на QR-вход: печатать **ASCII-QR в терминале** (`qr.print_ascii()` или
  `qrcode` с `print_ascii`) + ссылку `tg://login?token=…`; сканируешь телефоном.
  Дальше как раньше — генерит готовый блок для dockhost. Без кодов.

## Краевые случаи

- Не подтвердили за ~180 сек → сообщение «истекло», клиент отключён, визард сброшен.
- 2FA — поддержано (после подтверждения спрашиваем пароль).
- Отмена на любом шаге → `qr_task` отменяется, клиент отключается, временные файлы
  чистятся.
- Тапабельность `tg://login` зависит от клиента Telegram; не сработает тап — есть
  QR со второго устройства.
- Риск бана самого `api_id` остаётся, но к доставке кода уже не привязан.

## Тестирование

- Юнит-тестов на новый поток нет (всё на Telethon/QR). Существующие тесты
  (settings, bot_ui, profiles_dynamic) остаются зелёными; тест `extract_code`
  удаляется вместе с функцией.
- Ручная проверка: ➕ → ссылка/QR → подтверждение на телефоне → (2FA) → аккаунт
  поднялся; обновление QR по таймеру; отмена; переживание перезапуска.

## Что НЕ входит (YAGNI)

- Возврат ввода по коду (полностью убираем).
- Поддержка добавления без доступа к телефону аккаунта (невозможно в принципе).
