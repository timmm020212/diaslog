# Публичная кнопка «Подключиться» + вход на выбор (код / QR)

Дата: 2026-06-17

## Цель

Дать **любому** пользователю бота подключить **свой** аккаунт в систему перехвата
самостоятельно — кнопкой «🔌 Подключиться» под приветствием. При подключении
пользователь выбирает способ входа:
- **📱 По коду** — на одном телефоне (код приходит в его же Telegram), без второго экрана;
- **🖥 По QR** — без кода, но нужен второй экран для сканирования.

Перехваты подключённого аккаунта идут его владельцу (= самому этому аккаунту).

## Почему два способа

- Telegram даёт подтвердить login-токен (QR) **только сканом со второго устройства**
  — на одном телефоне нельзя (проверено: тап по ссылке/картинке не подтверждает).
- Код работает **на одном телефоне**: приходит в чат «Telegram» залогиненного
  аккаунта. Единственный однодевайсный способ.
- Поэтому даём выбор — каждый берёт под свою ситуацию.

## Поток (UX)

1. `/start` → у **всех** кнопка **🔌 Подключиться** (рядом с ⚙️ Настройки; у админа
   ещё 🛠 Админ-панель). Админская «➕ Добавить аккаунт» ведёт в тот же выбор.
2. Нажал «Подключиться» → экран выбора: **[📱 По коду] [🖥 По QR] [Отмена]**.
3. **По коду:** «Пришли номер телефона (с +)» → номер → «Код пришёл в твой Telegram
   (чат "Telegram"), введи его разбито: 1 2 3 4 5» → код (выпариваем цифры) →
   [2FA пароль если есть] → готово.
4. **По QR:** бот шлёт QR + инструкцию (скан со второго экрана) → подтверждение →
   [2FA] → готово.
5. Готово: «✅ Аккаунт N добавлен», слежка поднимается, перехваты идут владельцу
   (он уже в чате с ботом — нажал /start, открыв его).

Кнопка **Отмена** на любом шаге.

## Доступ и безопасность

- Публично: любой подключает **только свой** аккаунт (владелец = id залогиненного
  аккаунта). Чужой аккаунт подключить нельзя.
- Гейта нет. Минус — при массовом подключении одним общим `api_id` Telegram может
  зарезать выдачу кодов на ключ (QR это не затрагивает). Для узкого круга не проблема;
  при необходимости позже разнесём по разным api_id или добавим ограничение.

## Архитектура

### `admin.py` — `AccountManager`, визард на два режима
- `Wizard`: добавить `mode` ∈ {"code", "qr"}; поля `phone`, `phone_code_hash`
  (код) и `qr`, `qr_msg`, `qr_task` (QR). `step` ∈ {"phone","code","password"}
  (код) и {"qr","password"} (QR).
- `begin_code(user_id)`: создать клиент, connect, отправить «пришли номер»,
  `step="phone"`, `mode="code"`.
- `begin_qr(user_id)`: текущий QR-поток (qr_login + фоновая задача), `mode="qr"`.
- `feed_message(user_id, text)`: ветки по шагу:
  - `phone` → `send_code_request` → `step="code"`;
  - `code` → `extract_code` + `sign_in(phone,code,hash)`; `SessionPasswordNeededError`
    → `step="password"`; `PhoneCodeInvalid/Expired` → повтор;
  - `password` → `sign_in(password)` → финал.
  Ошибки `FloodWaitError`/прочее → cancel + понятный текст.
- `_finalize`/`_cleanup`/`cancel`/`start_profile`/`remove` — как сейчас (cancel
  гасит `qr_task`, если есть).
- Вернуть `extract_code(text)` (выпаривание цифр) + юнит-тест.

### `bot_ui.py`
- `welcome_buttons(is_admin)` → добавить ряд «🔌 Подключиться» (`CB_CONNECT`) всем.
- `connect_method_text()` + `connect_method_buttons()` → выбор [📱 По коду
  (`CB_CONN_CODE`)] / [🖥 По QR (`CB_CONN_QR`)] / [Отмена (`CB_ADMIN_CANCEL`)].
- `parse_admin` (или общий разбор) понимает `connect` / `conn:code` / `conn:qr`.
- Подпись QR (в `admin.py _qr_caption`) переписать **без** ссылки `tg://login`
  (она вводит в заблуждение): только инструкция «открой чат на втором устройстве и
  отсканируй QR через Настройки → Устройства → Подключить устройство».

### `run.py`
- `on_callback`: `connect` (любой) и admin `add` → показать экран выбора способа;
  `conn:code` → `manager.begin_code(sender_id)`; `conn:qr` → `manager.begin_qr(sender_id)`.
- `on_message`: снять гейт `is_admin` — пускать шаг ввода (номер/код/пароль) у
  **любого** пользователя с активным визардом (`sender_id in manager.wizards`,
  текст не команда).
- `_safe_answer` и admin-ветки — без изменений.

## Тестирование

- Юнит-тест: вернуть `extract_code` + тест на выпаривание цифр; `bot_ui` — разбор
  `connect`/`conn:code`/`conn:qr` и наличие кнопки «Подключиться» в welcome.
- Ручная проверка: «Подключиться» → По коду (один телефон, код в своём приложении)
  → вход; По QR (второй экран) → вход; 2FA; отмена; перехваты идут владельцу;
  переживание перезапуска; кнопка видна и работает у не-админа.

## Что НЕ входит (YAGNI)

- Гейт/инвайты на публичное подключение (позже при необходимости).
- Раздача отдельных api_id под нагрузку (позже).
- Удаление своего аккаунта пользователем (пока только админ удаляет).
