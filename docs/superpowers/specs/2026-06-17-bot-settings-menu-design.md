# Меню бота и настройки фильтрации (DIASLOG INTERCEPT)

Дата: 2026-06-17

## Цель

Дать каждому владельцу управлять своим перехватом прямо в чате с общим ботом:
синяя кнопка «Меню» → `/start` → приветствие + кнопка ⚙️ Настройки, где можно
включать/выключать бота и выбирать, какие типы событий присылать (удалённые,
изменённые, одноразовые) в любой комбинации.

Бот общий на всех; настройки — на каждого владельца отдельно (по `owner_id`),
поэтому друг меняет только своё, владелец — своё.

## Поведение фильтров: «не кэшировать отключённое»

Фильтры режут на уровне захвата, а не только доставки — отключённые типы не
сохраняются. Следствие, принятое осознанно: что произошло, пока тип был выключен,
задним числом не появится.

Чтобы поймать **удалённое** сообщение, его надо было сохранить при получении,
поэтому гейтинг каскадный (проверяется в `Capturer`):

- **`enabled == False`** (бот выключен): юзербот остаётся подключён (для мгновенного
  включения), но `_on_new` / `_on_deleted` / `_on_edited` сразу выходят. Ничего не
  сохраняется, не качается, не отправляется.
- **Сохранение входящих в кэх** (`store.save_message`) выполняется, если
  `deleted == True` ИЛИ `edited == True` (нужно обоим для детекта). Если оба `False`
  — входящие не пишутся.
- **Скачивание медиа входящих** (для пересылки удалённого фото/видео/стикера) —
  если `deleted == True` и `cache_media == True`.
- **Одноразовые (view-once)** обрабатываются (скачивание + отправка), если
  `view_once == True`; иначе игнорируются полностью (не качаются, не сохраняются).
- **`_on_edited`**: если `enabled == False` — выход. Если сообщение есть в кэше —
  всегда обновляем сохранённый текст (дёшево, держит «удалённые» актуальными); затем,
  если `edited == True`, шлём уведомление «было/стало».
- **`_on_deleted`**: если `enabled == False` или `deleted == False` — выход.

## Интерфейс бота

### Кнопка «Меню»
На старте `run.py` выставляет команды бота (`SetBotCommandsRequest` с
`BotCommandScopeDefault`, команда `start`). Telegram показывает синюю кнопку «Меню»
у поля ввода; по нажатию доступен `/start`.

### /start
Отвечает текущим приветствием (`DIASLOG INTERCEPT`, что приношу) + инлайн-кнопка
**⚙️ Настройки** под сообщением.

### Экран настроек
Кнопка «Настройки» редактирует то же сообщение (callback) в экран с переключателями:

```
⚙️  Настройки — аккаунт «timur»

Что присылать при перехвате:

[ 🟢 Бот включён ]                 ← вкл/выкл всё (🔴 когда выключен)
[ 🗑 Удалённые ✅ ] [ ✏️ Изменённые ✅ ]
[ 👁 Одноразовые ✅ ]
[ ◀️ Назад ]                        ← обратно к приветствию
```

- Галочка ✅/❌ на кнопке отражает текущее состояние; тап переключает и сразу
  перерисовывает клавиатуру (`event.edit`).
- Когда бот выключен — шапка показывает 🔴 и пометку, что перехватов нет; кнопки
  типов остаются доступными для настройки.
- «Назад» возвращает к приветствию с кнопкой ⚙️ Настройки.

## Архитектура

### `settings.py` (новый модуль)
Класс `Settings` на один аккаунт:
- Поля: `enabled`, `deleted`, `edited`, `view_once` (bool). По умолчанию все `True`
  — поведение как сейчас.
- `load(path)` / `save()` — JSON в `data_dir/settings.json` на диске `/data`.
  Отсутствие файла = дефолт (все включены).
- `toggle(key)` — переключить поле и сохранить.

### `profiles.py`
`Profile` получает путь `settings_path = data_dir/settings.json`.

### `capturer.py`
`Capturer.__init__(profile, store_factory, bot_client=None, settings=None)`.
Обработчики `_on_new` / `_on_deleted` / `_on_edited` сверяются с `self.settings`
по каскаду из раздела «Поведение фильтров».

### `run.py`
На старте:
1. Поднимает общий бот, выставляет команды (кнопка «Меню»).
2. Для каждого аккаунта грузит `Settings`, создаёт `Capturer(prof, Store,
   bot_client=bot, settings=settings)`.
3. Держит реестр `{owner_id: Settings}`.
4. Регистрирует обработчики бота:
   - `/start` → приветствие + кнопка ⚙️ Настройки.
   - `CallbackQuery` → разбор `data` (`open_settings` / `back` / `toggle:<key>`),
     поиск настроек по `event.sender_id`, изменение/сохранение, перерисовка.

Один и тот же объект `Settings` используется и капчурером, и обработчиком кнопок,
поэтому изменения вступают в силу мгновенно, без перезапуска.

## Краевые случаи

- `/start` или кнопки от не-владельца (его id не совпал ни с одним `owner_id`):
  приветствие показываем, в настройках — «к тебе не привязан аккаунт».
- Настройки переживают перезапуск контейнера (лежат на `/data`).
- Предполагаем, что `owner_id` уникален между аккаунтами (сейчас так: ты и друг).
- Дефолт для нового аккаунта — все типы включены (обратная совместимость).

## Что НЕ входит (YAGNI)

- Просмотр истории перехватов через бота.
- Настройка `RETENTION_DAYS` / `CACHE_MEDIA` из бота (остаются в `.env`).
- Раздельные фильтры по чатам/людям.
