# Diaslog Spy bot

Программа для **твоего** Telegram-аккаунта. Следит за **личными чатами и группами**
и присылает тебе в личку через бота:
- 🗑 удалённые сообщения (текст + медиа),
- ✏ изменённые (отредактированные) сообщения,
- 👁 одноразовые (view-once) фото/видео (только в личке).

> ⚠️ Работает только на аккаунтах, которые залогинены на этом компьютере, и только на
> своих переписках. Автоматизация аккаунта формально против правил Telegram — на свой риск.
> Программа должна быть запущена постоянно: пока она выключена, события не ловятся.

## Структура
- `run.py` — запуск всех аккаунтов 24/7 (это запускается в облаке/в Docker).
- `main.py` — терминальный режим, нужен для **первого входа** в аккаунт.
- `capturer.py` — логика захвата (юзербот + общий бот-доставщик), `store.py` — кэш для
  восстановления удалённых, `profiles.py` — аккаунты и конфиг бота, `util.py` — мелочи.

## Установка
```powershell
pip install -r requirements.txt
```

## Настройка
Бот **один на всех** (общий доставщик). Каждый аккаунт шлёт перехваты своему владельцу,
поэтому в одном боте у разных людей — только свои уведомления.

1. Создай бота у @BotFather (`/newbot`). Скопируй `.env.bot.example` → `.env.bot`,
   впиши `BOT_TOKEN`, а `API_ID` / `API_HASH` — любые валидные (можно те же, что у аккаунта).
2. Получи `api_id` / `api_hash` аккаунта на https://my.telegram.org → API development tools.
3. Скопируй `.env.example` → `.env`, впиши `API_ID`, `API_HASH`.
4. Узнай свой `OWNER_ID`: напиши общему боту `/start` — он пришлёт твой ID, впиши в `.env`.

Второй аккаунт (друг и т.п.) — то же самое в файле `.env.friend` (см. `.env.friend.example`),
но **без своего бота**: его `OWNER_ID` — это id друга, перехваты с его аккаунта пойдут ему.
Данные лягут в отдельную папку `data-friend/`.

## Первый вход (один раз, через терминал)
```powershell
python main.py            # профиль по умолчанию (.env)
python main.py friend     # профиль friend (.env.friend)
```
Введи номер телефона и код из Telegram. Сессия сохранится в `data/` (или `data-friend/`).
Пароль 2FA при вводе **не отображается** — это нормально, печатай вслепую и жми Enter.

После входа `main.py` уже сам ловит события и шлёт их боту. Можно так и оставить его
запущенным, либо закрыть и поднять все аккаунты разом через `run.py`.

## Повседневное использование
Закрой терминальные окна `main.py` (чтобы не было двойного входа) и запусти:
```powershell
python run.py
```
Поднимутся все аккаунты с сохранённой сессией. Перехваты приходят тебе в личку от бота
в реальном времени. Для облака есть `Dockerfile` / `docker-compose.yml` — см. `DEPLOY.md`.

## Настройки (.env)
| Параметр | Значение |
|---|---|
| `CACHE_MEDIA` | `true` — скачивать входящие медиа, чтобы присылать удалённые фото/видео |
| `RETENTION_DAYS` | сколько дней хранить кэш входящих (по умолчанию 7) |

## Данные
- `data/`, `data-friend/` — кэш (`cache.db`), медиа (`media/`) и сессии (`*.session`).
- Сессии = доступ к аккаунту. **Никому не передавай.** Не попадают в git (см. `.gitignore`).

## Безопасность
- Программа **не запрашивает коды входа** в фоне — только вручную через `main.py`.
- Перехваты приходят лично владельцу (`OWNER_ID`) — наружу ничего не публикуется.
